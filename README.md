# NSFC 基金申请书智能撰写系统 (4+1 Agent 协作模式)

## 📌 系统定位与目标

本项目旨在解决传统基金申请撰写中流程繁琐、易有疏漏的问题。通过引入大模型与智能协作架构，将研究者的核心想法自动转化为符合 NSFC（国家自然科学基金）标准的正式申请书文档，一举跳过文献整理、框架搭建、格式调整等繁琐步骤。

## 🧠 核心架构：4+1 Agent 协作模式

系统采用基于 **LangGraph** 的核心 Orchestrator（协调器），统筹把控整体进度并协调四位各司其职的 AI 专家（Agent）协同工作：

1. 👑 **Writer Agent (首席研究员)**：
   - 整合专家意见，主导撰写并生成最终申请书内容初稿。
2. 🔍 **Searcher Agent (文献专员)**：
   - 自动提取研究关键词。
   - 接入 arXiv 获取最新文献，同时支持查询本地私有学术知识库（RAG），为申请书（尤其是立项依据）提供坚实支撑。
3. ⚖️ **Reviewer Agent (科学评论家)**：
   - 对生成的初稿进行严肃的学术审查，自动输出结构化的修订建议与评估打分。
4. 🎨 **Designer Agent (基金策略师)**：
   - 从宏观与排版角度优化内容结构、逻辑连贯性，并致力于拔高项目的创新性表达。

## ⚙️ 实现与工作流程

本系统的整个撰写过程被封装为自动化的标准工作流：

1. **文献检索**：基于用户输入的研究主题，自动检索相关前沿文献和本地积累。
2. **分层讨论**：由 Writer Agent 主持，搜集其他专家 Agent 提供的背景资料与策略建议，融合生成申请书初稿。
3. **自动评审**：初步内容生成后，唤醒 Reviewer Agent 挑刺并输出修改补丁。
4. **内联修订确认**：提供清晰直观的交互界面，研究者可以像使用 Word 修订模式一样，逐条查看、对比文本差异（**<span style="color:red">红删</span><span style="color:green">绿增</span>**），自主决定是否接受修订请求。所有操作具有完整的可追溯性。

## 💡 技术亮点

- **智能权重分配**：系统能解析 NSFC 模板，动静态结合识别章节（例如“立项依据”作为核心章节，将分配更高的推理算力和更严格的输出标准）。
- **差异化对比引擎 (Diff Engine)**：精确定位文本更改内容，支持复杂的长文本比对展现。
- **多模型支持与降级链 (Fallback Chain)**：无缝支持 DeepSeek、Kimi、豆包等国内外主流大模型。系统内置容错机制，当主选模型限流或不可用时，无感切换至备用模型，保证业务中不中断。
- **完整审计追踪**：工作流的所有请求响应、Agent 内部思考路径将全程写入运行日志，过程透明、有据可查。

## 🚀 具体使用说明（使用范式）

本系统倡导新的科研与 AI 协作范式：**由 AI 团队承担繁重的内容起草、结构决策，研究者则抽身退步，专注于高层次的科学指导和最终成果的把关。**

### 环境配置

在使用系统前，需要配置 `.env` 文件以设置相关模型的 API 密钥：

```bash
DEEPSEEK_API_KEY="your_api_key_here"
KIMI_API_KEY="your_api_key_here"
DOUBAO_API_KEY="your_api_key_here"
```

### 准备工作

将最新的 NSFC 标准空白模板文件放入系统的 `templates/` 目录中。系统会自动读取并拆解各章节规定。

### 启动与撰写

1. 在网页界面中选择**项目类型**（如面上项目、青年基金等）。
2. 输入您的**研究主题**及核心思路。
3. 点击 **“开始会议”**。系统将展示各 Agent 的交流过程。
4. 待初稿和审核完成后，进入**内联修订界面**对细节进行微调与确认。

### 导出与终稿

审核完成后，点击导出，系统将即刻生成符合最终 NSFC 格式规定、排版精美的 Word/PDF 文档，可直接用于申报。`uvicorn backend.main:app --reload --port 8000`
